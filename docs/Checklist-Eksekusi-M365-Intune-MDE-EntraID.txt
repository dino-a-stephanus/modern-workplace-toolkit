HARI 2 — ENTRA ID, MFA, CONDITIONAL ACCESS
=====================================================================

[ ] SECURITY GROUPS
    [ ] Login ke Entra admin center: https://entra.microsoft.com
    [ ] Buat security group: All-Staff (masukkan semua 10 user)
    [ ] Buat security group: Management (masukkan user sesuai jabatan)
    [ ] Buat security group: IT-Admin (untuk scope policy khusus admin)

[ ] AKTIFKAN MFA
    [ ] Entra admin center → Identity → Users → Per-user MFA
    [ ] Aktifkan MFA untuk semua 10 user sekaligus
    [ ] Atau: gunakan Security Defaults jika tidak butuh Conditional Access custom
        (Entra → Overview → Properties → Manage Security Defaults)

[ ] CONDITIONAL ACCESS POLICY
    [ ] Buat policy 1: Block Legacy Authentication
        - Users: All users
        - Conditions: Client apps → Legacy authentication clients
        - Grant: Block access
    [ ] Buat policy 2: Require MFA for All Users
        - Users: All users (exclude break-glass account)
        - Cloud apps: All cloud apps
        - Grant: Require MFA
    [ ] Buat policy 3: Require Compliant Device
        - Users: All users
        - Grant: Require device to be marked as compliant
        - (Aktifkan setelah Intune enrollment selesai di Hari 3)
    [ ] Buat break-glass account (emergency access) — exclude dari semua CA policy
        [ ] Simpan kredensial break-glass di tempat aman (offline)

[ ] ADMIN HARDENING
    [ ] Pastikan Global Admin hanya digunakan untuk task admin, bukan email/Teams sehari-hari
    [ ] Aktifkan MFA untuk admin account
    [ ] Review assigned roles — pastikan tidak ada user dengan role berlebih

HARI 3 — MICROSOFT INTUNE (DEVICE MANAGEMENT)

    [ ] Set MDM Authority ke Microsoft Intune

        - MDM user scope: All
        - MAM user scope: All

[ ] COMPLIANCE POLICY
    [ ] Buat compliance policy untuk Windows:
        [ ] Require BitLocker encryption
        [ ] Require secure boot
        [ ] Minimum OS version (Windows 10/11 sesuai device klien)
        [ ] Require password: min 8 karakter, kompleksitas aktif
        [ ] Maximum minutes of inactivity before password required: 15 menit
    [ ] Buat compliance policy untuk iOS/Android (jika ada device mobile):
        [ ] Require device encryption
        [ ] Minimum OS version
        [ ] Require PIN

HARI 4 — MICROSOFT DEFENDER FOR ENDPOINT (MDE)
=====================================================================

[ ] SETUP MDE
    [ ] Login ke Microsoft Defender portal: https://security.microsoft.com
    [ ] Settings → Endpoints → Onboarding → pilih metode onboarding
        (untuk Windows via Intune: gunakan Endpoint security → Endpoint detection and response)

[ ] ONBOARDING DEVICE VIA INTUNE
    [ ] Intune admin center → Endpoint security → Endpoint detection and response
    [ ] Buat policy EDR:
        - Platform: Windows 10 and later
        - Profile: Endpoint detection and response
        - Microsoft Defender for Endpoint client configuration package type: Auto from connector
    [ ] Assign ke group All-Staff
    [ ] Verifikasi semua 10 device onboard di Defender portal → Assets → Devices
        [ ] Device 1: ________ ✓/✗
        [ ] Device 2: ________ ✓/✗
        [ ] Device 3: ________ ✓/✗
        [ ] Device 4: ________ ✓/✗
        [ ] Device 5: ________ ✓/✗
        [ ] Device 6: ________ ✓/✗
        [ ] Device 7: ________ ✓/✗
        [ ] Device 8: ________ ✓/✗
        [ ] Device 9: ________ ✓/✗
        [ ] Device 10: ________ ✓/✗

[ ] KONFIGURASI MDE
    [ ] Setup Attack Surface Reduction (ASR) rules — mode Audit dulu (bukan Block)
        (Intune → Endpoint security → Attack surface reduction)
    [ ] Aktifkan Cloud-delivered protection & Automatic sample submission
    [ ] Konfigurasi alert notification:
        Settings → Endpoints → Email notifications → Add notification rule
        (kirim alert ke email admin untuk severity: Medium, High, Critical)
    [ ] Review Secure Score di Defender portal — catat baseline score awal

[ ] INTEGRASI MDE + INTUNE (CONDITIONAL ACCESS)
    [ ] Defender portal → Settings → Endpoints → Advanced features
        [ ] Aktifkan: Microsoft Intune connection → On
    [ ] Intune → Tenant administration → Connectors → Microsoft Defender for Endpoint
        [ ] Pastikan status: Enabled
    [ ] Aktifkan Conditional Access policy Hari 2 (Require Compliant Device)

- Simpan break-glass account credentials offline (bukan di cloud)
- ASR rules: mulai dari mode Audit 1-2 minggu sebelum ganti ke Block,
  supaya tidak ada false positive yang ganggu kerjaan karyawan
- DMARC: mulai dari p=none (monitoring), setelah 2-4 minggu cek report,
  baru naikkan ke p=quarantine atau p=reject
