DAY 2 — ENTRA ID, MFA, CONDITIONAL ACCESS
=====================================================================

[ ] SECURITY GROUPS
    [ ] Log in to Entra admin center: https://entra.microsoft.com

    [ ] Enable MFA for all 10 users at once

        (Entra → Overview → Properties → Manage Security Defaults)

[ ] CONDITIONAL ACCESS POLICY
    [ ] Create policy 1: Block Legacy Authentication
        - Users: All users
        - Conditions: Client apps → Legacy authentication clients
        - Grant: Block access
    [ ] Create policy 2: Require MFA for All Users

        - Cloud apps: All cloud apps

    [ ] Create policy 3: Require Compliant Device
        - Users: All users

    [ ] Create a break-glass account (emergency access) — exclude from all CA policies

IMPORTANT NOTES
- Never send user credentials via a WhatsApp group — use a private message or an encrypted document
- Store break-glass account credentials offline (not in the cloud)
- ASR rules: start in Audit mode for 1-2 weeks before switching to Block,
  to avoid false positives disrupting employees' work
- DMARC: start with p=none (monitoring), review reports after 2-4 weeks,
  then raise it to p=quarantine or p=reject
- Always revoke admin access once the project is complete and handover is done
